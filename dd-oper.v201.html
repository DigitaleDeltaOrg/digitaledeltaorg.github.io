http://api.rws.nl/dl/

Digital Delta - Operational API - version 2.0.1

Introduction

The Digital Delta is a distributed network in which various data sources exchange information using a standard web service interface. The architecture and additional description of the Digital Delta can be found on www.digitaledelta.org. The present document contains the specification of the Digital Delta web service API. It uses the RAML standard as the definition language.

The API specification

This API describes 'DD-OPER', which is a series of additional endpoints to the 'DD API'. The structure of the /timeseries response of DD-OPER API is in line with the /timeseries response defined by DD-API, but will provide a different (sub)set of the information.

get

get /dd-oper/2.0/quantities/{quantityName}/locations

Request the list of locations (within the NL.RWS.WM namespace) at which a specific quantity is available.

  • Request
  • Response

URI Parameters

  • quantityName: required(string)

    A name referencing a category of time series. The quantity names are predefined. For example 'waterlevel', 'watertemperature', 'waveheight'. Remark: Only used in DD-OPER

    Example:

    waterLevel

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • provider: (object)

    Information regarding the system that is responsible for the response. Provides support contact information in case there are issues with response format or content.

    • name: (string)

    • supportUrl: (string)

      Url to information for the provider (of the service).

    • apiVersion: (string)

      API Version with which the result set was generated. Can be used to parse the response.

    • responseType: (string)

      Name of the type of the result set. Can be used to parse the response.

    • responseTimestamp: (string - minLength: 24)

      Timestamp when this response was created.

      Example:

      2018-07-05T15:11:36.123Z,  2018-07-05T15:11:36.123456Z, 2018-07-05T15:11:36.000Z
  • results: required(array of DD-T.Location)

    Items: Location

    • type: required(Feature)

      Type of GeoJSON object. In the DD-API the only allowed type is "Feature".

    • geometry: required(union of DD-T.Geometry or nil)

      The coordinate(s) of the Location are placed within the "geometry" section of the GeoJSON object. DD-API uses Point and Multi-Point locations, but is not restricted to these two.

    • properties: (object)

      The attributes of the Location object are placed within the properties group of the GeoJSON object.

      • url: (string)

        Url describing the location object. (Not provided in case of /dd-oper requests.)

      • node: (object)

        Specification of the node where the location is stored.

        • id: (string)

          Id of the node.

        • name: required(string)

          The name of the node.

        • description: (string)

          The description of the node.

        • baseUrl: (string)

          The base URL of the node.

      • locationId: (string)

        Id of the location.

      • locationCode: (string)

        Example:

        HVH
      • locationName: (string)

        Name of the location.

        Example:

        Hoek-van-Holland
      • referenceLevel: (string)

        GeoJSON element. Specifies the reference plane for the altitude coordinates (third). For example "NAP".

      • crsName: (string)

        The coordinate reference system of the given coordinates according to the INSPIRE definition (see url https://inspire.ec.europa.eu/documents/Data_Specifications/INSPIRE_Specification_CRS_v3.0.pdf section 4.4.1) GeoJSON does nolonger support reference to a Coordinate Reference System. It assumes coordinates are defined according WGS84. However, within DD-API there is a need to provide coordinates in different coordinate-systems, for instance ETRS89. The element is optional and when omitted WGS84 should be assumed. Remark: INSPIRE does not define 'ETRS89' as such, but specifies different 'flavors' of it with a suffix. To prevent unnecessary translation the commonly used value 'ETRS89' is allowed as equivelant for the INSPIRE value 'ETRS89-XYZ'.

      • locationNameSpace: (string)

        Remark: Only used in DD-OPER response.

        Example:

        NL.RWS.WM
      • displayNameDetail: (string)

        Remark: Only used in DD-OPER response.

        Example:

        Hoek-van-Holland
      • displayNameGlobal: (string)

        Remark: Only used in DD-OPER response.

        Example:

        Hoek-van-Holland
      • displayNameSpace: (string)

        Remark: Only used in DD-OPER response.

        Example:

        NL.RWS.WM
      • parentName: (string)

        Remark: Only used in DD-OPER response.

        Example:

        Hoek-van-Holland

Example:

{
	"provider": {
		"name": "MKK-DL",
		"supportUrl": "tbd http://www.rws.nl/support/mkk-dl",
		"responseType": "LocationListResponse",
		"responseTimestamp": "2018-07-05T15:11:36.123456Z"
	},
	"results": [
		{
			"type": "Feature",
			"geometry": {
				"type": "Point",
				"coordinates": [
					4.123456,
					52.123456
				]
			},
			"properties": {
				"locationName": "maarssen.oostkanaaldijk",
				"locationNameSpace": "NL.RWS.WM",
				"displayNameGlobal": "Maarssen",
				"displayNameDetail": "Oostkanaaldijk",
				"parentName": "noordzeekanaal-amsterdamrijnkanaal",
				"referenceLevel": "NAP",
				"crsName": "ETRS89"
			}
		},
		{
			"type": "Feature",
			"geometry": {
				"type": "MultiPoint",
				"coordinates": [
					[
						4.123456,
						52.123456
					],
					[
						4.123456,
						52.123456,
						-1.0

			},

				"displayNameGlobal": "Maarssen",
				"displayNameDetail": "Westkanaaldijk",
