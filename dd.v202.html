Digital Delta API version 2.0.2

Introduction

The Digital Delta is a distributed network in which various data sources exchange information using a standard web service interface. The architecture and additional description of the Digital Delta can be found on www.digitaledelta.org. The present document contains the specification of the Digital Delta REST service API. It uses the RAML standard as the definition language.

/dd/2.0/nodes get

get

A (data-)node is an area from which resource objects can be retrieved. A node may have to access multiple sources to obtain information.

get /dd/2.0/nodes

A (data-)node is an area from which resource objects can be retrieved. A node may have to access multiple sources to obtain information.

  • Request
  • Response

Query Parameters

  • process: (string)

    The name of the mechanism to collect/derive the time series value. When accessing RWS' DD-API, providing a processName as query parameter is required. The process names are pre-defined (see examples). Extending the processname with a suffix, referencing a more detailed mechanism like a specific calculation model, is not allowed. Other implementions dot need to implement this query-parameter.

    Example:

    measurement, forecast, astronomical, advise
  • name: (string)

    (Partial) name of the node. Partial meaning startsWith(...).

    Example:

    PrimaryNode
  • page: (integer)

    Page number, starting with 1.

  • pageSize: (integer)

    Requested response Page size. Minimum, maximum and default is defined by Provider. Minimum can never be lower than 1.

Body

Media type: application/json

Type: object

Properties
  • provider: (object)

    Information regarding the system that is responsible for the response. Provides support contact information in case there are issues with response format or content.

    • name: (string)

      Name of the provider (of the service).

    • supportUrl: (string)

      Url to information for the provider (of the service).

    • apiVersion: (string)

      API Version with which the result set was generated. Can be used to parse the response.

    • responseType: (string)

      Name of the type of the result set. Can be used to parse the response.

    • responseTimestamp: (string - minLength: 24)

      Timestamp when this response was created.

      Example:

      2018-07-05T15:11:36.123Z,  2018-07-05T15:11:36.123456Z, 2018-07-05T15:11:36.000Z
  • paging: (object)

    The object contains urls and page size information to support back and forward paging

    • totalObjectCount: (integer)

      The actual number of objects available. When paging is used the response contains a subset of the total obejcts available up to the max page size.

    • maxPageSize: (integer - minimum: 1 - maximum: 10000)

      Maximum number of objects in the response. Defined by the Provider of the response.

      Example:

      10
    • minPageSize: (integer - minimum: 1 - maximum: 10000)

      Minimum number of objects in the response. Defined by the Provider of the response.

      Example:

      10
    • self: (union of DD-T.Url or nil)

      Url of the current request.

    • first: (union of DD-T.Url or nil)

      Url to the first page of the result set.

    • prev: required(union of DD-T.Url or nil)

      Url to the previous page of the result set.

    • next: required(union of DD-T.Url or nil)

      Url to the next page of the result set.

    • last: (union of DD-T.Url or nil)

      Url to the last page of the result set.

  • results: required(array of DD-T.Node)

    List of Node objects

    Items: Node

    • id: (string)

      Id of the node.

    • name: required(string)

      The name of the node.

    • description: (string)

      The description of the node.

    • baseUrl: (string)

      The base URL of the node.

TimeSeriesType objects describe the type of measurement that the timeseries contains. These can be, for example, raw measurements, but also aggregations or simulations. This information could be needed to correctly interpret the measurement values. A request where both startTime and endTime are omitted returns a response with just the metadata.
